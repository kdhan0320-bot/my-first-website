# Mini SNS

작업 기록과 스터디 모임을 공유하는 모바일 소셜 앱 데모입니다.

- 배포 주소: https://kdhan0320-bot.github.io/my-first-website/mini-sns/

---

## 제작 목적

모바일 SNS 앱의 핵심 화면 흐름을 구현하기 위해 제작한 프로젝트입니다.

홈 피드, 게시글 작성, 프로필, 모임, 채팅, 알림 화면을 하단 탭 내비게이션으로 연결해 사용자가 여러 기능을 자연스럽게 이동할 수 있도록 구성했습니다.

Portfolio Feedback Hub가 포트폴리오 피드백 게시판이라면, Mini SNS는 모바일 앱의 화면 전환과 소셜 기능 흐름을 보여주는 프로젝트입니다.

---

## 주요 기능

- 게스트 모드
- 테스트 계정 체험 안내
- 홈 피드
- 게시글 작성 화면
- 피드 이미지 확대 모달
- 프로필 화면
- 모임 목록
- 모임 참가 후 채팅방 이동
- 채팅 목록 / 채팅방 상세
- 메시지 입력창 하단 고정
- 알림 화면
- 하단 탭 내비게이션
- 좋아요 / 댓글 UI
- 반응형 모바일 앱 레이아웃

---

## 테스트 계정

- 화면 입력 아이디: demo
- 실제 Supabase Auth 이메일: demo@minisns.app
- 비밀번호: demo1234!

테스트 계정은 Supabase Auth Dashboard에서 직접 생성해야 합니다.

테스트 계정 로그인이 실패해도 게스트 모드로 홈 피드, 모임, 채팅, 알림, 프로필 등 주요 화면을 확인할 수 있습니다.

## 데모 데이터

이 프로젝트는 취업용 포트폴리오 데모이므로 실제 운영 서비스가 아닙니다.

샘플 게시글, 프로필, 모임, 채팅, 알림은 작업 기록과 스터디 모임 흐름을 보여주기 위해 구성했습니다.

실제 Supabase 프로젝트(profiles/posts/comments/likes/follows 테이블)를 정리하거나 데모 계정을 준비해야 한다면 `supabase/reset-demo-data.sql` 파일을 참고하세요. 모임/채팅/알림 화면은 Supabase 없이 프론트엔드 목(mock) 데이터로 구성되어 있어 별도 DB 세팅이 필요 없습니다. `auth.users` 계정 생성/삭제는 이 저장소의 코드로 자동화하지 않으며, Supabase Dashboard 또는 Admin API에서 직접 처리해야 합니다.
