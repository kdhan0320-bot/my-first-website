# AI 협업 프로토콜

## 1. 목적과 적용 범위
이 문서는 `my_ai_web` 저장소에서 ChatGPT, VS Code Claude, 사용자가 같은
작업 기준을 공유하도록 정리한 협업 절차입니다. 저장소 전체에 적용됩니다.

작업 규칙은 작업 대상과 가장 가까운 CLAUDE.md가 루트 CLAUDE.md보다
구체적인 기준으로 적용됩니다.

README.md와 관련 docs는 프로젝트의 목적·구현·실행 방법을 확인하는
사실 자료이며, CLAUDE.md 작업 규칙을 대체하거나 덮어쓰지 않습니다.

## 2. 저장소 전체 기준과 포트폴리오 전용 기준 구분
저장소 전체는 취업 포트폴리오 전용 폴더가 아니라 AI/UX/UI 작업실이며,
공용 기준은 루트 `CLAUDE.md`가 관리합니다. 취업용 표현·대표 프로젝트
선별·문장 규칙 등 포트폴리오 전용 기준은 `projects/my-portfolio/CLAUDE.md`
에서 별도로 관리하며, 루트 문서에 복제하지 않습니다.

## 3. 지시와 자료의 우선순위

### 작업 지시 우선순위

1. 현재 사용자가 명시한 작업 목적과 승인 범위
2. 작업 대상과 가장 가까운 `CLAUDE.md`
3. 루트 `CLAUDE.md`
4. `docs/ai-workflow/AI_COLLABORATION_PROTOCOL.md`

README와 관련 docs는 작업 지시가 아니라 사실 확인 자료로 사용합니다.

사용자 지시가 보안, 데이터 손실, 기존 승인 규칙과 충돌하면 임의로 실행하지
않고 충돌 내용을 보고합니다.

### 사실 확인 우선순위

1. 현재 로컬 파일과 Git 상태
2. 연결된 GitHub·Supabase 등에서 현재 확인한 상태
3. ChatGPT가 확인 주체와 근거를 명시해 전달한 사실
4. Claude의 이전 보고서와 과거 감사 자료

현재 검증 가능한 자료와 과거 보고가 충돌하면 현재 자료를 우선합니다.
확인할 수 없는 내용은 `확인 불가`로 표시합니다.

## 4. ChatGPT 역할
- GitHub 원격 상태(커밋, 브랜치, Actions 실행 결과), Supabase 프로젝트·
  Advisor 등 연결 도구로 확인 가능한 운영 상태, 공식 문서 등 웹 자료를
  확인합니다.
- `tools/site-audit-kit`이 생성한 점검 보고서 ZIP을 검토합니다.
- Claude에게 작업을 인수인계할 때 이 문서 7번 항목의 형식을 따릅니다.

## 5. VS Code Claude 역할
- 로컬 전체 파일을 검색하고 실제 파일을 수정합니다.
- import·경로 참조, build, lint, Playwright 등 로컬에서 실행 가능한 검사를
  담당합니다.
- 현재 진행 중인 ChatGPT 대화를 자동으로 알 수 없으므로, 프롬프트에 전달된
  인수인계 정보와 사용자 승인 범위만을 기준으로 작업합니다.

## 6. 사용자 역할
- 파일·폴더 삭제, 이동, 이름 변경을 승인합니다.
- 디자인 최종안, 결제, 계정 설정을 결정합니다.
- git commit, push, PR 생성, merge, deploy를 승인합니다.
- ChatGPT와 Claude 사이에서 판단이 갈릴 때 최종 결정을 내립니다.

## 7. ChatGPT에서 Claude로 전달할 정보
ChatGPT가 Claude에게 작업을 인수인계할 때는 다음을 구분해서 전달합니다.

- ChatGPT가 독립적으로 확인한 사실
- Claude 보고에만 의존하는 사실
- 사용자가 승인한 내용
- 현재 기준 branch와 commit
- 이번 작업 범위
- 정확한 변경 대상 파일
- 변경 금지 범위
- 검사 명령과 성공 조건
- 중단 조건
- commit·push·deploy 허용 여부

## 8. 조사·수정·검사·commit·push·deploy 단계 분리
작업은 조사 → 수정 → 검사 → (승인 시) commit·push·deploy 순서로 단계를
나눕니다. 각 단계 사이에 예상과 다른 상태(Git 기준 불일치, 참조 파일 없음
등)를 발견하면 다음 단계로 넘어가지 않고 중단 보고합니다.

## 9. 중복 검사가 허용되는 경우
ChatGPT가 이미 확인한 내용이라도, 로컬 파일·코드·Git 상태와 직접 관련된
사실은 Claude가 다시 확인합니다.

Claude가 직접 재검증할 수 없는 GitHub 원격, Supabase 운영 상태, 공식 웹
자료는 ChatGPT가 확인한 인수인계 사실로 사용할 수 있습니다.

이 경우 최종 보고에서 확인 주체가 ChatGPT임을 구분하고, Claude가 직접
확인한 것처럼 표현하지 않습니다.

ChatGPT의 인수인계 내용과 현재 로컬 자료가 충돌하면 임의로 한쪽을 선택하지
않고 중단 보고합니다.

## 10. 작업 종류별 검사 선택

```text
문서 수정:
참조 검색, 문장 정합성, git diff --check

코드 수정:
build, lint, 관련 기능 검사

UI 수정:
지정 viewport, 가로 스크롤, 겹침, 잘림, 콘솔 오류

배포 수정:
workflow, artifact, Actions, 운영 URL

Supabase 수정:
기존 migration, 운영 적용 여부, Advisor, RLS

자산 수정:
참조 경로, 라이선스, 배포 포함 여부
```

## 11. 사실성·API·DB·mock·static·메모리 상태 구분
프로젝트 설명은 실제 데이터 방식을 그대로 표현합니다. Supabase Auth·DB에
실제로 연동된 프로젝트, 게스트 모드로 샘플 데이터를 보여주는 프로젝트,
mock 데이터와 브라우저 메모리 상태로만 동작하는 프로젝트, 정적
HTML/CSS/JS로 동작하며 백엔드가 없는 프로젝트를 서로 혼동해 표현하지
않습니다.

## 12. 중단 조건
다음 중 하나라도 해당하면 수정을 진행하지 않고 중단 보고합니다.

- 기준 Git 상태(branch, HEAD, origin 비교, working tree)가 예상과 다름
- 인수인계에 언급된 파일이나 경로가 로컬에 존재하지 않음
- 변경 대상이 확실하지 않거나 변경 금지 범위와 겹침
- 사용자 승인 없이 삭제·이동·commit·push·deploy가 필요해 보임
