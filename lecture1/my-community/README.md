# Portfolio Feedback Hub

취업 준비생과 UX/UI 학습자가 포트폴리오를 공유하고 피드백을 주고받을 수 있는 커뮤니티형 게시판입니다.

배포 주소: https://kdhan0320-bot.github.io/my-first-website/my-community/

---

## 제작 목적

포트폴리오를 만들었지만 객관적인 피드백을 받기 어려운 문제를 해결하기 위해, 카테고리별 피드백 요청과 댓글 중심의 커뮤니티 흐름을 구현했습니다. "기능이 많은 일반 게시판"이 아니라 포트폴리오 피드백이라는 목적에 맞춘 정보 위계와 상태 배지, 피드백 요청 템플릿을 중심으로 설계했습니다.

---

## 주요 기능

- **로그인 / 회원가입** — Supabase Auth 기반 아이디/비밀번호 로그인
- **게스트로 둘러보기 / 테스트 계정으로 체험하기** — 회원가입 없이 게스트 모드로 탐색하거나, 데모 계정으로 실제 로그인 흐름 체험 가능
- **게시글 목록 카드형 그리드** — 데스크톱 3열/태블릿 2열/모바일 1열 반응형 카드
- **카테고리별 기본 썸네일** — 카테고리마다 고유 아이콘/그라디언트로 구성된 기본 썸네일을 카드에 표시해 주제를 빠르게 구분
- **카테고리/상태 배지** — 카테고리 라벨과 "피드백 요청중 / 답변 완료 / 취업 준비 / AI 활용" 상태 배지 표시
- **카테고리 필터 + 정렬** — 아이콘이 있는 카테고리 칩 필터, 최신순/인기순/댓글 많은 순/조회수 순 정렬
- **게시글 검색** — 제목, 내용, 해시태그 통합 검색
- **게시글 작성 / 수정 / 삭제** — Supabase 데이터 연동
- **피드백 요청 템플릿** — 글쓰기 시 "색상/여백/정보구조/기능 흐름/취업용 문구/반응형/접근성/코드 구조" 중 피드백 받고 싶은 부분을 선택해 본문에 자동 반영
- **랜덤 이미지 첨부** — 게시글 상세/작성 화면에서 이미지 추가 가능 (목록 카드의 썸네일은 카테고리 기본 썸네일을 사용)
- **해시태그** — 최대 5개 태그 추가 및 검색/카테고리 연동
- **댓글 / 대댓글** — 2단계 댓글 구조 구현
- **댓글 좋아요 / 게시글 좋아요** — 실시간 반영, 비로그인 상태에서는 로그인 안내
- **조회수** — 게시글 상세 접속 시 자동 카운트
- **샘플 데이터 기반 데모 화면** — 6개 카테고리별 샘플 게시글/댓글로 빈 화면 없이 시연 가능

| 분류 | 기술 |

## UX/UI 개선 포인트

- 헤더(AppBar)와 본문 컨테이너의 가로 정렬을 `Container maxWidth="lg"` + `Toolbar disableGutters`로 통일
- 랜덤 이미지 대신 카테고리별 기본 썸네일(아이콘 + 그라디언트)을 적용해 게시글의 주제를 빠르게 파악할 수 있도록 개선
- 카드 내 정보 위계를 상태/카테고리 배지 → 제목 → 요약 → 해시태그 → 작성자/지표 순으로 정리하고, hover 시 카드가 살짝 떠오르는 효과 추가
- 첫 화면 Hero 영역에 서로 다른 카테고리의 미니 게시글 프리뷰(클릭 불가)를 배치해 서비스 목적을 시각적으로 전달
- 하단 링크를 텍스트 화살표 링크 대신 "Project Info" 카드 + 버튼(GitHub 보기 / 포트폴리오로 돌아가기) 구조로 정리
- 게스트 모드 및 테스트 계정으로 로그인 없이 주요 화면 탐색 가능
- 게시물 상세 페이지는 "목록으로 돌아가기"만 남기고, 포트폴리오 링크는 목록 페이지 하단으로 분리
- 게스트 상태에서 글쓰기/좋아요 시도 시 에러 대신 로그인 안내 제공
- 모바일 환경을 고려한 반응형 레이아웃 (헤더, 카드, 검색/필터 영역)

## 테스트 계정

- 아이디: `demo`
- 비밀번호: `demo1234!`
- 로그인 페이지의 "테스트 계정으로 체험하기" 버튼으로 바로 로그인할 수 있습니다.
- 관리자 권한은 없으며, 일반 사용자 권한(RLS 적용)만 가집니다. 타인의 글/댓글을 삭제할 수 없습니다.
- 별도 계정 없이 둘러보고 싶다면 "게스트로 둘러보기"를 이용해주세요. 이 프로젝트는 데모 목적의 프로젝트이며, 실제 개인정보를 입력하지 않는 것을 권장합니다.

## 실행 방법

```bash
# 의존성 설치
npm install

# 개발 서버 실행 (http://localhost:5173)
npm run dev

# 프로덕션 빌드
npm run build
```

배포는 `main` 브랜치에 push되면 저장소 루트의 GitHub Actions 워크플로(`.github/workflows/deploy.yml`)가 자동으로 빌드 후 GitHub Pages에 배포합니다.
