# Portfolio Feedback Hub

취업 준비생과 UX/UI 학습자가 포트폴리오를 공유하고 피드백을 주고받을 수 있는 커뮤니티형 게시판입니다.

* 배포 주소: https://kdhan0320-bot.github.io/my-first-website/my-community/

---

## 제작 목적

포트폴리오를 만들었지만 객관적인 피드백을 받기 어려운 문제를 해결하기 위해, 카테고리별 피드백 요청과 댓글 중심의 커뮤니티 흐름을 구현했습니다.

단순히 기능이 많은 게시판이 아니라, 포트폴리오 피드백이라는 목적에 맞춘 정보 위계와 상태 배지, 피드백 요청 템플릿을 중심으로 설계했습니다.

---

## 주요 기능

* 로그인 / 회원가입 (Supabase Auth 기반)
* 게스트로 둘러보기
* 테스트 계정 체험
* 게시글 목록 카드형 그리드 (데스크톱 3열 / 태블릿 2열 / 모바일 1열)
* 카테고리별 기본 썸네일
* 히어로 미니 프리뷰
* Project Info 섹션
* 카테고리/상태 배지 ("피드백 요청중 / 답변 완료 / 취업 준비 / AI 활용")
* 카테고리 필터 + 정렬 (최신순 / 인기순 / 댓글 많은 순 / 조회수 순)
* 게시글 검색 (제목, 내용, 해시태그)
* 게시글 작성 / 수정 / 삭제
* 피드백 요청 템플릿 (색상 / 여백 / 정보구조 / 기능 흐름 / 취업용 문구 / 반응형 / 접근성 / 코드 구조)
* 랜덤 이미지 첨부 (게시글 상세/작성 화면 전용, 목록 카드 썸네일은 카테고리 기본 썸네일 사용)
* 해시태그 (최대 5개)
* 댓글 / 대댓글
* 댓글 좋아요 / 게시글 좋아요
* 조회수
* 샘플 데이터 기반 데모 화면 (6개 카테고리별 샘플 게시글/댓글)

## UX/UI 개선 포인트

* 헤더(AppBar)와 본문 컨테이너의 가로 정렬을 `Container maxWidth="lg"` + `Toolbar disableGutters`로 통일했습니다.
* 랜덤 이미지 대신 카테고리별 기본 썸네일(아이콘 + 그라디언트)을 적용해 게시글의 주제를 빠르게 파악할 수 있도록 했습니다.
* 카드 내 정보 위계를 상태/카테고리 배지 → 제목 → 요약 → 해시태그 → 작성자/지표 순으로 정리하고, hover 시 카드가 살짝 떠오르는 효과를 추가했습니다.
* 첫 화면 히어로 영역에 서로 다른 카테고리의 미니 게시글 프리뷰(클릭 불가)를 배치해 서비스 목적을 시각적으로 전달합니다.
* 하단 링크를 텍스트 화살표 링크 대신 "Project Info" 카드 + 버튼(GitHub 보기 / 포트폴리오로 돌아가기) 구조로 정리했습니다.
* 모바일 화면에서는 히어로 CTA 버튼 문구를 "데모 체험하기"로 축약해 줄바꿈 없이 표시됩니다.
* 상태 배지 색상은 옅은 배경 위에서도 WCAG AA 기준(4.5:1) 이상의 텍스트 대비를 갖도록 조정했습니다.
* 게시물 상세 페이지는 "목록으로 돌아가기"만 남기고, 포트폴리오 링크는 목록 페이지 하단으로 분리했습니다.
* 게스트 상태에서 글쓰기/좋아요/댓글 시도 시 에러 대신 로그인 안내를 제공합니다.
* 모바일 환경을 고려한 반응형 레이아웃(헤더, 카드, 검색/필터 영역)을 적용했습니다.

## 테스트 계정

* 아이디: `demo`
* 비밀번호: `demo1234!`

배포 환경의 Supabase Auth에 등록되어 있는 일반 사용자 계정입니다. 로그인 페이지의 "테스트 계정으로 체험하기" 버튼을 누르면 로그인 후 글쓰기·댓글·좋아요 기능을 체험할 수 있습니다. 관리자 권한은 없으며, 타인의 글/댓글은 삭제할 수 없습니다.

Supabase 프로젝트 설정 상태에 따라 테스트 계정 로그인이 일시적으로 실패할 수 있습니다. 이 경우 화면에 실패 안내가 표시되며, "게스트로 둘러보기"를 통해 목록, 상세, 검색, 필터, 정렬 등 주요 화면은 계속 확인할 수 있습니다.

이 프로젝트는 데모 목적의 프로젝트이며, 실제 개인정보를 입력하지 않는 것을 권장합니다.

## 실행 방법

```bash
npm install
npm run dev
npm run build
```

배포는 `main` 브랜치에 push되면 저장소 루트의 GitHub Actions 워크플로(`.github/workflows/deploy.yml`)가 자동으로 빌드 후 GitHub Pages에 배포합니다.

---

## 환경 변수

Supabase 연동을 위해 아래 환경 변수가 필요합니다. `.env.example`을 참고해 로컬에 `.env`를 만들어 사용하세요.

```env
VITE_SUPABASE_URL=your_supabase_project_url
VITE_SUPABASE_ANON_KEY=your_supabase_anon_key
```

GitHub Actions 배포 시에는 저장소 Secrets(`SUPABASE_URL`, `SUPABASE_ANON_KEY`)에서 값을 주입받습니다. `src/lib/supabase.js`에는 로컬 환경 변수가 없을 때를 대비한 개발용 fallback 값이 남아있으며, 이는 현재 배포가 의존하고 있어 임의로 제거하지 않았습니다.
